# Weather Computer — Protocol Status

This document tracks the implementation status of the Block 1 protocol and related components.

The architecture and protocol decisions referenced here come from the project design discussions.  
Future revisions may change details, but this document reflects the current implementation state.

---

# Block 1 Overview

Block 1 establishes the baseline architecture for the Weather Computer system.

Core design goals:

- Provide situational weather awareness while driving.
- Continue estimating conditions for up to ~2 hours after losing cellular connectivity.
- Use a stored regional weather field and resample it as the vehicle moves.

# Protocol Implementation Status

## Python Reference Implementation

Status: COMPLETE

Location:
weather_protocol/

Capabilities:

- Packet encoding
- Packet decoding
- CRC validation
- Strict protocol validation
- Fixture generation
- Human-readable packet dump tool

Supported packet types:

- RegionalSnapshotV1
- PositionUpdateV1
- AckV1

Binary fixtures are generated for testing and firmware integration.

---

## Rust Firmware Parser

Status: COMPLETE

The Rust parser will:

- parse the common packet header
- validate magic, version, length, and CRC
- decode packet payloads
- apply strict semantic validation
- reject malformed packets

The Rust implementation must match the Python reference implementation exactly.

Fixture files generated by Python will be used for Rust parser tests.

Current state:

- common header parsing implemented
- strict magic, version, length, and CRC validation implemented
- RegionalSnapshotV1, PositionUpdateV1, and AckV1 decoding implemented
- fixture-based Rust tests passing against Python-generated binary fixtures

---

## Weather Interpolation Engine

Status: IMPLEMENTED

Responsibilities:

- interpolate local weather conditions from the regional grid
- estimate conditions as the vehicle moves
- degrade confidence as forecast age increases

Confidence is computed on-device only and is not transmitted in packets.

Current state:

- Block 1 spatial and temporal interpolation implemented in Rust
- local confidence scoring implemented on-device only
- Rust interpolation unit tests passing for center, edge, outside-field, stale-position, beyond-horizon, and hazard behavior
